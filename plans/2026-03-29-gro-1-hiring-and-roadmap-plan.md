# GRO-1 Hiring And Roadmap Plan

Date: 2026-03-29
Issue: GRO-1

## Objective

Hire the first engineer, define a narrow MVP for the growth-hacking product, and create the first execution queue so work can start immediately after onboarding.

## Hiring Plan

1. Hire a Founding Engineer who reports to the CEO and owns technical execution.
2. Use the existing managed project workspace as the engineer's default working directory.
3. On approval, assign the engineer the first implementation tasks and use them as the initial technical owner for the product.

## MVP Scope

The first milestone should stay narrow:

- collect structured answers through a guided Q&A flow
- turn those answers into a concrete growth plan
- present suggested companies, SaaS tools, and ideas tied to the user's answers

## Initial Task Breakdown

1. Define the MVP technical architecture, data flow, and acceptance criteria for the Q&A-to-plan pipeline.
2. Implement the interactive Q&A experience and state handling for a first usable vertical slice.
3. Implement plan generation and recommendation output for companies, SaaS tools, and tactics.

## Risks

- The engineer hire may wait on board approval before execution can begin.
- Product scope is broad; the first milestone needs strict MVP discipline.
- There is no existing code in the managed workspace, so the first engineer will need to establish the initial application structure.

## Immediate Next Step

Submit the Founding Engineer hire request and attach the plan to the issue so the board can review the operating approach.
